# Contributing

1. Do not commit `.env`, passwords, PLAXIS project files, or licensed executables.
2. Preserve all PPL notices in vendor and derivative files.
3. Document changes to PLAXIS-derived routines in source comments and pull requests.
4. Run `python scripts/check_repository.py` before committing.
5. Treat changes to phase, polygon, plate, interface, and node identifiers as scientific-model changes requiring PLAXIS validation.
